# Contributing

## Branching

- `main` protegida.
- Trabajar en ramas:
  - `feature/<scope>`
  - `fix/<scope>`
  - `chore/<scope>`

## Commit Convention

Usar Conventional Commits:

- `feat: ...`
- `fix: ...`
- `chore: ...`
- `docs: ...`
- `refactor: ...`
- `test: ...`

Ejemplo:
`feat(orgchart): add conflict queue and resolution actions`

## Pull Requests

- PR obligatorio para merge a `main`.
- Al menos 1 aprobación.
- CI en verde.
- Completar template de PR.
